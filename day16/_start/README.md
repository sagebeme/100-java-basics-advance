### Initial State
When you start Day 16, you're likely:
- **Familiar** with procedural programming from Days 1-15
- **Ready** to learn OOP concepts
- **Curious** about classes, objects, and encapsulation
- **Unsure** about when and how to use OOP

### Common Starting Challenges

1. **OOP Paradigm Shift**
   - Thinking in objects vs procedures
   - Understanding classes as blueprints
   - Grasping object instantiation
   - **Tip**: Think of classes as templates, objects as instances

2. **Encapsulation Concepts**
   - Private vs public access
   - Why hide data?
   - Getters and setters
   - **Tip**: Private protects data, public methods control access

3. **Class Design**
   - What belongs in a class?
   - How to organize code?
   - Single Responsibility Principle
   - **Tip**: Each class should have one clear purpose

4. **Constructor Confusion**
   - What are constructors?
   - When to use them?
   - Parameterized vs default
   - **Tip**: Constructors initialize object state

### Time Expectations

- **First time**: 4-5 hours (OOP is a big concept!)
- **OOP concepts**: Takes time to internalize
- **Class design**: Practice helps with organization

### Key Concepts to Grasp

- **Classes are blueprints** - Define structure and behavior
- **Objects are instances** - Created from classes
- **Encapsulation** - Hide data, expose methods
- **this keyword** - Reference to current object

### Before Moving Forward

Make sure you can:
- [ ] Create classes with attributes and methods
- [ ] Write constructors
- [ ] Understand encapsulation
- [ ] Create and use objects
